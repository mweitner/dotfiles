# AI / Automation Guidelines for dotfiles

Purpose: Provide concise rules for AI agents and automated tooling that modify files in this repository.

Scope:

- Edits to shell scripts in `shell/`, service snippets in `systemd/`, config files under `sway/`, `i3/`, `fish/`, and documentation files.

Rules:

- Keep changes minimal and focused. Prefer one logical change per commit.
- Do not invent credentials, secrets, or private hostnames. If a secret is required, add a clear placeholder and instruct the maintainer to populate it from a secure store.
- Follow repository style: Markdown line length <= 120 characters.
- Validate shell changes with `shellcheck` and `shfmt` where possible before proposing changes.
- For any change that modifies installer scripts (`install-*.sh`) or system-wide configuration, include a short manual verification checklist in the commit message or PR description.
- All AI-generated edits must be human-reviewed before merge.
- Add tests or lint fixes where practical (for example, add `shfmt` formatting changes in the same commit that introduces other shell edits).

Operations that require explicit human approval (do not do automatically):

- Adding new `sudo` commands, new systemd service files, or changes that write to `/etc` paths.
- Adding external package repositories or importing GPG keys.

Contact: Refer to repository `CODEOWNERS` or repo maintainers for approvals and sensitive changes.
